# SA Architect Guide

Внутренний гайд: путь системного аналитика к роли солюшн архитектора.
Единый базовый стандарт (21 цель в 4 этапах), стартовая диагностика, артефакты-доказательства и ревью этапов руководителем.

Статическое приложение (один `index.html`) + Supabase для авторизации и хранения. Сервера нет.

## Запуск
1. Настрой Supabase по `SETUP.md` (таблицы, RLS, аккаунты).
2. Впиши `SUPABASE_URL` и `SUPABASE_ANON_KEY` в начало `<script>` в `index.html`.
3. Захости (GitHub Pages — см. `GITHUB_DEPLOY.md`).

## Роли
- Аналитик: проходит диагностику, закрывает цели, прикладывает артефакты, отправляет этапы на ревью.
- Ревьюер (в таблице `reviewers`): видит прогресс команды, принимает или возвращает этапы.

anon-ключ Supabase безопасно держать в браузере — доступ защищает Row Level Security.
